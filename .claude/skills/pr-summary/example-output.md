Refactor vector DB product ingestion script for dev/prod environments

<BODY>
## Summary

Refactored the product data ingestion script to support both development and production environments with improved configuration management and better error handling.

## Changes

### 1. Script Refactoring (`services/shopping-assistant/scripts/ingest_product_data_into_vectorstore.py`)

#### Environment-Based Configuration
- Added `--env` argument with choices `["dev", "prod"]` to support environment-specific configuration
- Implemented dynamic `.env` file loading based on environment

```python
if app_env == "prod":
    app_env_files = [str(REPO_ROOT / "platform/app/.env")]
elif app_env == "dev":
    app_env_files = [
        str(REPO_ROOT / "platform/app/.env"),
        str(REPO_ROOT / "platform/app/.env.dev"),
    ]
```

#### Improved Path Handling
- Introduced `REPO_ROOT` constant using `Path(__file__).parents[CUR_SCRIPT_REPO_ROOT_DEPTH]`
- Changed `--sqlite-db-path` to `--ecom-backend-db-sqlite-path` with default value
- All paths now resolved relative to repository root

```python
REPO_ROOT = Path(__file__).parents[CUR_SCRIPT_REPO_ROOT_DEPTH]
ecom_backend_db_sqlite_path = str(REPO_ROOT / args.ecom_backend_db_sqlite_path)
```

### 2. Makefile Updates

Simplified the `ingest-products-vectordb` target:

```bash
# Before
ingest-products-vectordb:
	python chatbot/scripts/ingest_product_data_into_vectorstore.py \
	--sqlite-db-path "ecom-backend/ecom_backend.db"

# After
ingest-products-vectordb:
	uv run --python services/shopping-assistant/.venv \
	services/shopping-assistant/scripts/ingest_product_data_into_vectorstore.py
```

### 3. Docker Compose Configuration (`platform/app/docker-compose.yml`)

Fixed hardcoded Weaviate host configuration:

```yaml
# Before
WEAVIATE_HTTP_HOST: weaviate
WEAVIATE_GRPC_HOST: weaviate

# After
WEAVIATE_HTTP_HOST: ${WEAVIATE_HTTP_HOST}
WEAVIATE_GRPC_HOST: ${WEAVIATE_GRPC_HOST}
```

## Benefits

1. **Environment Flexibility**: Script now supports both dev and prod configurations without code changes
2. **Better Portability**: All paths resolved relative to repository root
3. **Improved Reliability**: Validation ensures consistent Weaviate configuration
4. **Enhanced Maintainability**: Clear separation of concerns with constants and validation logic

## Testing

To test the refactored script:

```bash
# Run in dev mode (default)
make ingest-products-vectordb

# Or explicitly specify environment
uv run --python services/shopping-assistant/.venv \
  services/shopping-assistant/scripts/ingest_product_data_into_vectorstore.py \
  --env dev
```
